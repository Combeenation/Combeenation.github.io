Embedding a configurator

This page should give you an overview on how to embed a Combeenation configurator into your own website.

Quick start

The embed code for each configurator will be shown in the gallery. Basically you just need to paste the code on the desired place on your website and you're good to go.

If you are using a CMS (e.g. WordPress, Drupal, ...) there is most likely some kind of HTML editor, which allows you to add the embed code to the page. If you are unsure where to put the code or don't have access to your code please contact your web administrator.

How does the embedding work?

Configurators are embedded with the help of an iframe, this HTML 5 element basically just displays another website (e.g. Combeenation configurator) in the current website (e.g. your website).

To see how this works you can click on the "Show embed example" button in the gallery. This will open up a new page where you can take a look at the source code to see how the embedding works in a full page.

Embed code in detail

As already mentioned the embed code can be found in the gallery and contains the name of your company and configurator and looks simliar to the following one: 

<script src="https://portal.combeenation.com/plugin/YOURCOMPANY/YOURCONFIGURATOR"></script>
<iframe class="cbn-configframe" frameborder="0" src="https://portal.combeenation.com/Cfgr/YOURCOMPANY/YOURCONFIGURATOR"></iframe>

The first element (<script>) which will be included is the Combeenation plugin. The plugin enables the communication between the iframe and the parent page via cross document messaging (Wikipedia, MDN web docs). The communication is needed for e.g. passing the checkout data to the parent page or reopening the configurator after a product has been shared on social media.

The second html element is the <iframe> which contains the Combeenation configurator, which can be placed anywhere on the page. The iframe already has the CSS class cbn-configframe set, which shouldn't be removed, because it is used by the plugin to identify the configurator. There are several properties in the iframe which may be useful and can be modified:

  • src ... address of the document which should be embedded. By default it contains the copy URL of the configurator, which means that every user which enters the page starts with a new configuration. If you want a user to edit an existing configuration you can simply set the edit URL for the iframe. A list of useful URLs can be found here
  • frameborder ... Specifies whether or not to display a border around the iframe (default: 0)
  • height ... Height of the iframe
  • width ... Width of the iframe

Whitelisting

Combeenation also allows you to set whitelisted domains for you configurator. This gives you the power to display the configurator only on your whitelisted domains, this way others (e.g. competitors) can't embed the configurator on their page.

The whitelisted domains will be enforced with the help of the Content-Security-Policy header, which will be set for your configurator.

Configurator settings

The whitelisted urls can be edited under 'Configurator Settings' in the configurators overview. Every change will be immediatly active.

  • Add: Write the desired url into the textbox and press Enter or click 'Add URL'
  • Edit: Doubleclick on the entry and submit with Enter
  • Delete: Click the minus-icon on the right side and confirm the dialog that opens

Configurator URLs

Each URL starts with https://portal.combeenation.com/Cfgr and can end with ...

  • /{YourCompany}/{YourConfigurator} ... the default configurator URL which will start a new configuration based on the default preset
  • /{YourCompany}/{YourConfigurator}/{PresetName} ... configurator URL which will start a new configuration based on the given preset
  • /{YourCompany}/{YourConfigurator}/{ConfigurationId}/copy ... URL which starts a new configuration based on a given configuration
  • /{YourCompany}/{YourConfigurator}/{ConfigurationId}/edit/{ConfigurationAuthToken} ... URL which lets the user edit an existing configuration

Here is an explanation of the substitutes (terms in curly braces {})

  • {YourCompany} ... the reference name for your company
  • {YourConfigurator} ... the reference name for your configurator
  • {PresetName} ... the id or the name of your preset (see preset view in configurator editor, where you can also set the default preset for your configurator)
  • {ConfigurationId} ... is a unique number which identifies a configuration, will be generated for each new configuration.
  • {ConfigurationAuthToken} ... text which is needed to edit an existing configuration, will be generated for each new configuration.
