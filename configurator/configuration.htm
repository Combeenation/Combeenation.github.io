Configuration

If you are interested on how to interact with the configuration in Hive check out the Current Configuration page.

A configuration is created each time a user opens a configurator and is used to save the user's progress. This is done by saving all the user input values, which are used to compute the result values. The configuration is identified by a unique id and can be edited by using the auth token.

Shared configuration

If a configuration is shared on a social media a so called shared configuration is created. Besides some additional meta data needed for the social media platform, this is basically a normal configuration.

Finish

A Finish is created when a checkout is performed. Besides the input values, the result values are stored in a finish. Those values can be used by the company for further processing (e.g. production). Each finish can also contain a price, which will be set via the shop property price. The price will be used for the calculation of the Total Revenue in the metrics overview.

Metrics

On June 2021 metrics have been introduced. Those metrics will be displayed for each published configurator. By default the last 7 days will be viewed, but the range query can be adjusted and even compared with older time ranges.

Dashboard metrics

Quick overview over all the metrics provided in the dashboard:

Metric Detail
Configurations Number of configurations + finishes (finishes count will be added since those were configurations before the checkout)
Finishes Number of finishes
Shared Number of shared configurations
Total Revenue Is the sum of all finish prices (Can be zero if no price is set for a finish)
Conversion Rate Defines the ratio between created configurations and finishes Finishes / Configurations * 100
