General information about the Configurator

This page should explain you the different parts of the configurator.

Presets

Presets are used to start the configurator with defined values.

Example: At the customers website are different points to open the configurator. At one point you have a grey and round window. At another point you have a green window with a milky glass inside. Each point has different start values, therefore you can create presets.

Components and groups

At the left side of your configurator you will find a list of components and groups.
Each component is part of a group. For a better overview in your configurator, split your components in useful groups.

Always use meaningful and unique names for your components and their groups.

To add a component, use the + at the top.

To add a group you first have to add a component and during this process you are able to add the component to a new group.

Signal & Slots

Signals & Slots are used to connect components and controls.
“SigSlo” is the short form of it.
To add SigSlo's you can use the button at the right top of the Design Page or you click on a control and open the SigSlo window from there.

Example:

If you click on a button, a value of a component should be changed.

  • Signal: click on the button
  • Slot: change the value

Two Way Binding

If you create a SigSlo always think if there is a SigSlo in the other way back. If the value of the component can be change at any other point, has it any effect on the button?

Design component

Add the right side you find the elements of your design component, the properties and SigSlo's. For styling your controls try to use the properties in the list. It is easier to find it and understand it. If you have to set other properties you can give your control a CSS class and style it in the “CSS Style” Page.

For elements which are used at more than one point and always styled the same way, you can use a CSS class is stead of always changing the same properties each time the element occurs.

Escape Operator

Each component has the following properties:

  • group
  • name

To access the value of this properties you use the # operator, the name of the component and the name of the property.

Example:

  • #RecColor.Group
    • name of the group which contains the record component “RecColor"
    • in this case "Color"
  • #RecColor.Name
    • contains the name of the component
    • in this case "RecColor"

More properties for different types of components:

Record Component

  • #[NameoftheRecordComponent].Records: contains the whole record table

    Tipp

    You have a product with a top color and a bottom color. Both have the same selection of colors. You fill the first record component with your color table. In the second record component you write #RecColor.Records and you have the same color table. If a new color is necessary, you only have to add it in the first record component.

Specifically, to access any property of a RecordComponent, you have to use the Escape Operator. For example, to access the Records property you have to write #myComponent.Records. On the plus side, myComponent.Name will now always refer to the "Name" column (if there is one), meaning you don't have to write myComponent.Value.Name.

Style Guide

Record Component

For a better overview keep your table well structured. Use blanks instead of tabs. You will be happy about it, if you have to add a column later.

Naming Guide

CSS Classes

First write the name of the container, add two underscore and the name of the element:
OuterContainer__InnerElement

Example:
InfoPanel__DownloadButton
InfoPanel__HeaderLabel

For more information ask Google about "BEM - Block Element Modifier"

Formatted Text

Start and end your text with three instead of one quotation mark. So you don’t have to excape another quotation mark in the text or in the html-formating text.

To include a value of another component in the text, you use {= [NameOfTheComponent] }

Example: """ Das ist der Name der RecordComponent: „{=#RecTopTyp.Name}“!"""
Shown Result: Das ist der Name der RecordComponent:"RecTopTyp"!
