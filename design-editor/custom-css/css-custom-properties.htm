CSS custom properties (CSS Variables)

All browsers which are officially supported by Combeenation also support CSS custom properties. Therefore you can basically use them as you wish in your custom CSS.

See Can I use for more details on browser support.

Organizational tip

It is recommended to declare all CSS variables in a dedicated custom CSS module named Variables or something the like instead of spreading them all over different modules. This helps you keep an overview of all available variables.

Changing variable values at runtime

The values of CSS variables declared in custom CSS can be changed dynamically at runtime using the following SigSlo:

CssVars.ValueChanged -> Configurator.updateCssVariables(CssVars.Value)

The value which is given to the updateCssVariables slot needs to be the value of a record component with column names representing the names of the CSS variable without the leading --.

Warning!

If you want to change CSS variable values dynamically via SigSlo you have to use names which are also allowed in Hive. E.g. you couldn't use names like label-text-color but would have to stick to labelTextColor etc.

Example

Configurator CssVariables in company Combeenation College:
Published demo

/* Custom CSS */
:root {
    --cfgrBckgrColor: #DD0060;
    --labelTextColor: yellow;
    --labelBckgrColor: #37A58C;
}

.cfgr-parent {
    background: var(--cfgrBckgrColor) !important;
}

.cfgr-label {
    color: var(--labelTextColor);
    background: var(--labelBckgrColor);
}

/* Record component "CssVars" */
{
    { Theme=Text, cfgrBckgrColor=Text, labelTextColor=Text, labelBckgrColor=Text }
    { "Colorful", "#DD0060",           "yellow",            "#37A58C" }
    { "Light",    "white",             "white",             "rgba(0, 0, 0, 0.6)" }
    { "Dark",     "black",             "black",             "rgba(255, 255, 255, 0.6)" }
}

Support for legacy browsers (IE 11 etc.)

If you have to support older browsers like IE 11 you can still use CSS variables since Combeenation comes with a css-vars-ponyfill which provides support for them in legacy browsers.

However, be aware that the ponyfill comes with some limitations, the most important one being that you can only declare CSS variables within a :root ruleset but not within other selectors.

Warning!

When using CSS variables in your custom CSS, you should always pay close attention to the correct behavior in different browsers but even more so if you have to support legacy browsers such as IE 11.

Warning!

Even though Edge 15 & 16 basically support CSS variables, they come with some known issues. Therefore we're also using the ponyfill when on those browsers which means that the mentioned limitations apply to them as well. However, the market share of Edge 15 & 16 is quite low and should therefore be negligible.

Limitation example

Whilst the following would work in modern browsers, legacy browsers won't apply the overwritten color but show all text in blue:

:root {
    /* Default text color */
    --label-text-color: blue;
}

.RedTextPanel {
    /* Overwrite default text color for everything within a ".RedTextPanel" */
    --label-text-color: red;
}

.cfgr-label {
    color: var(--label-text-color);
}
